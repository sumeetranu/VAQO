1 Landing Page and Navigation

The landing page asks where you want your database data to come from: the test database, or an import from an external database.

The navigation bar consists of 5 elements:


  1. VAQO: This button routes you to the home/landing page.
  2. About: General information about VAQO and its capabilities.
  3. User Guide: Detailed user guide with guidance on every portion of the website.
  4. FAQ: A list of Frequently Asked Questions and their answers.
  5. Contact: Contact information for the VAQO development team.

2 Use Test Database

This option allows you to use the default test database when running queries. The test database cannot be modified by the user. Selecting this option will directly take you to the Workspace page where you can start performing operations on the dataset.

If you want to view the test data while in the Workspace tab, click the "Database" button on the right to view its contents.

3 Import External Database

This option allows you to import your own database from a SQL dump. Clicking on the "Import External Database" section will take you to a page where you can enter your SQL dump. Once you enter the SQL dump in the editor, click "Import" to import the database.

The image below shows a sample SQL dump that creates a table called "testsample", defines its schema and then inserts 3 rows of data.


If you want to modify the database at any point, click on the "Database" button on the Workspace page. This will take you back to the Import External Database page where you can enter the new SQL dump.

4 Workspace

The Workspace is where you will spend most of your time on the website. In the workspace, you can run relational algebra queries, optimize relational algebra queries and convert between relational algebra and SQL. The run and optimize tabs also show you a tree with the query break down.

The image below shows the major components of the Workspace page.


You can toggle between the three tabs: Run, Optimize and Convert. The Database button allows you to view the test database if you chose to use the test database or allows you to change the imported SQL dump if you chose to import the database.

4.1 Run

Create a relational algebra query using the buttons and text box fields above the run button. Then, you can hit the Run button to run the query on the current database.

When you run the query, you will see the resulting data table on the right side, and a query break down tree right below it. You can hover over the tree nodes to view additional info on the different sections. The image below shows the result of running a query.

4.2 Optimize

In the optimize tab, you can enter a relational algebra query into the text box. After that, simply hit "Optimize" to optimize the entered query.

After optimizing, you will see the new optimized query show up on the right, with two trees below it. The Original tree represents the query breakdown for the query you originally entered and the Optimized tree will show the new, optimized one.


4.3 Convert

The convert tab allows you to convert queries between Relational Algebra and SQL.

4.3.1 Relational Algebra to SQL

In order to convert from Relational Algebra to SQL, enter a Relational Algebra query in the box on the left. After entering the query, click the "Convert to SQL" button right below the box. An equivalent SQL query will appear in the box on the right.

The image below demonstrates where to enter Relational Algebra queries to convert to SQL.


4.3.2 SQL to Relational Algebra

In order to convert from SQL to Relational Algebra, enter a SQL query in the box on the right. After entering the query, click the "Convert to Relational Algebra" button right below the box. An equivalent Relational Algebra query will appear in the box on the left.
